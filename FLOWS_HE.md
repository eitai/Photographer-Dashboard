# מערכת הזמנות, ספק, חיוב ומסמכים — תיאור כל הזרימות

מסמך סקירה לבדיקה עצמית. מתאר את כל התהליכים במערכת: שלושת התפקידים, שלוש זרימות ההזמנה, ניהול הספק, החיוב החודשי, ההתחשבנות עם הספק, והקבלות/חשבוניות.

---

## תפקידים במערכת

| תפקיד | מי זה | איך נכנס |
|---|---|---|
| **סופר אדמין** | בעל הפלטפורמה (אתה) | `/admin` (פורטל ניהול) |
| **צלם (אדמין)** | צלם שמנהל גלריות ולקוחות | `/login` (דף הבית) |
| **לקוח** | בעל גלריה שמזמין מוצרים | קישור גלריה פרטי (בלי הרשמה) |
| **ספק** | מעבדת ההדפסות הבלעדית | `/supplier/login` |

**עיקרון מרכזי בכסף:** הפלטפורמה היא **הגובה היחיד והמשלם היחיד**. הלקוחות משלמים לפלטפורמה (מחיר קמעונאי), הצלמים משלמים לפלטפורמה (מחיר עלות), והפלטפורמה משלמת לספק פעם בחודש. אין עמלה לצלם כרגע.

---

## הגדרת מוצרים (אצל הספק)

הספק מגדיר לכל מוצר:
- שם, סוג (הדפסה / קנבס / אלבום / דיגיטלי / אחר), תיאור, מק"ט
- **מחיר עלות** (מה שהפלטפורמה משלמת לספק) ו**מחיר ללקוח** (קמעונאי)
- **מינימום ומקסימום תמונות** (0 = ללא הגבלה)
- **ימי ייצור**
- **וריאציות** (למשל: גודל → 20x30, 30x40)
- תמונת מוצר, פעיל/לא פעיל

הצלם **מסמן בכוכב מוצרים מועדפים** מקטלוג הספק (בהגדרות → פרופיל הסטודיו → "המוצרים שלי"). המועדפים מופיעים ראשונים ביצירת הזמנה.

**אכיפת כמות תמונות** נאכפת בכל מקום — גם בצד הלקוח (כפתורים נחסמים, טקסט דרישה באדום-ענבר) וגם בשרת (422 אם מפרים). לדוגמה אלבום עם מינימום 2 / מקסימום 4 → אי אפשר לשלוח עם תמונה אחת.

---

## זרימה 1 — הזמנה ביוזמת הצלם (הלקוח בוחר תמונות)

מתי: הצלם רוצה שהלקוח יבחר תמונות למוצר מסוים.

1. הצלם נכנס לעמוד הלקוח → "הזמנה חדשה" → בוחר מוצר + כמות + וריאציות.
2. המערכת מייצרת **קישור בחירה ייחודי** ושולחת ללקוח (מייל + SMS).
3. הלקוח פותח את הקישור (`/order-selection/:token`), בוחר את התמונות הנדרשות, ממלא כתובת משלוח, ומאשר.
4. הצלם מקבל התראה, **מאשר** את הבחירה.
5. הצלם **שולח לספק** → ההזמנה עוברת לסטטוס "נשלח לספק".
6. הצלם מקבל **אישור הזמנה** במייל.

---

## זרימה 2 — הזמנה עצמאית של הלקוח (חנות בגלריה)

מתי: לקוח רוצה לקנות מוצרים בעצמו מתוך הגלריה שלו.

1. הלקוח נכנס לגלריה → לשונית **"חנות"** (מופיעה רק אם הצלם מורשה — ראה הרשאות).
2. בוחר מוצר → בוחר תמונות (לפי דרישת המינימום/מקסימום) → בוחר וריאציות → מוסיף לעגלה.
3. ממלא כתובת משלוח ומבצע **תשלום קמעונאי דרך PayPlus**.
4. עם אישור התשלום (webhook):
   - ההזמנה עוברת אוטומטית ל"נשלח לספק" (ספק בלעדי).
   - הצלם מקבל התראה.
   - הלקוח מקבל **אישור הזמנה** + **קבלה** במייל.
5. הצלם **לא מעורב בתשלום** ולא מקבל עמלה (כרגע).

---

## זרימה 3 — הזמנה ישירה של הצלם (חנות הצלם)

מתי: הצלם מזמין מוצר ישירות, עם תמונות משלו.

1. הצלם נכנס ל**"חנות"** (`/admin/store`) — קטלוג המוצרים, מועדפים ראשונים.
2. בוחר מוצר → כמות + וריאציות.
3. **מקור התמונות** (שתי לשוניות):
   - **העלאת תמונות** (גרירה/בחירה) — נשמרות בגלריית-מערכת מוסתרת.
   - **מהגלריות שלי** — בוחר תמונות מכל גלריה שלו (אפשר לערבב).
4. **כתובת משלוח** (בחירה): כתובת הסטודיו שלי / כתובת לקוח / כתובת אחרת.
5. שולח → ההזמנה נוצרת ועוברת **ישר ל"נשלח לספק"** (בלי שלב בחירה/אישור — הצלם כבר בחר).
6. הצלם מקבל **אישור הזמנה** על כל הזמנה כזו.

> דרישה לפני הזמנה (זרימות 1 ו-3): לצלם חייב להיות **כרטיס אשראי שמור** והרשאת הזמנה, והוא לא חסום בגלל חוב. אחרת מופיע באנר חוסם עם קישור למסך החיוב.

## צינור הסטטוסים של הזמנה

```
טיוטה → ממתין לבחירה → בחירה הוגשה → מאושר → נשלח לספק
→ בייצור → מוכן למשלוח → נשלח → סופק
```
בכל שלב אפשר לבטל (עד שנשלח/מוכן למשלוח). הספק מעדכן: בייצור → מוכן למשלוח → נשלח (עם מספר מעקב) → סופק. ניתן גם לדלג מ"בייצור" ישר ל"נשלח".

**התראות אוטומטיות:** ללקוח על קישור בחירה, על אישור תשלום, ועל שינוי סטטוס (נשלח/סופק כולל מעקב). לצלם על הגשת בחירה, תשלום, ושינויי סטטוס מהספק. לספק על הזמנה חדשה.

## פאנל הספק

- **לוח בקרה** — כרטיסי מונים (חדשות / בייצור / מוכן למשלוח / נשלח / נמסר) + הזמנות אחרונות.
- **הזמנות** — רשימה עם סינון; הספק רואה **רק** הזמנות שנשלחו אליו (לא טיוטות/לפני שליחה).
- **פרטי הזמנה** — לקוח/כתובת, פריטים + וריאציות (כתגיות), תמונות שנבחרו, **"הורד תמונות"**, וטופס עדכון סטטוס + מספר מעקב.
- **מוצרים** — ניהול הקטלוג (כולל מינ'/מקס' תמונות, ימי ייצור, וריאציות).
- **התחשבנות** — קריאה בלבד: כמה הפלטפורמה חייבת לו + היסטוריית תשלומים.

בהזמנה ישירה (בלי לקוח) הספק רואה את שם הסטודיו ככתובת המשלוח.

## חיוב הצלמים (חודשי)

- הצלם צובר **מחיר עלות** על כל הזמנה שהוא מבצע (זרימות 1 ו-3). זרימה 2 לא נספרת (הלקוח כבר שילם).
- **חיוב חודשי** אוטומטי ב-1 לחודש (cron): נסגר המחזור הקודם, נוצרת חשבונית אחת לכל צלם, ומחויב הכרטיס השמור. יש גם כפתור ידני לסופר אדמין ("סגירת מחזור וחיוב").
- **כרטיס שמור** — חובה לפני הזמנה. הצלם מוסיף כרטיס פעם אחת (מאובטח אצל PayPlus, אנחנו שומרים רק token + 4 ספרות).
- **כישלון חיוב** → הצלם נחסם **מהזמנות שלו עצמו בלבד**. הלקוחות שלו ממשיכים להזמין מהחנות (Flow B) ולשלם ישירות. החסימה משתחררת כשהחוב מוסדר.
- **מסך גבייה** (סופר אדמין, `/admin/collection`): לכל צלם — הצטברות נוכחית, חשבונית אחרונה, כרטיס, חסום או לא. פעולות: סמן כשולם, בטל חסימה, סגור מחזור עכשיו.
- הצלם רואה במסך "חיוב הזמנות מהספק" (`/admin/billing-store`): כרטיס שמור, הצטברות החודש, וחשבוניות.

## התחשבנות מול הספק (חודשי)

- הפלטפורמה משלמת לספק **בבולק פעם בחודש** (העברה בנקאית ידנית).
- **מסך התחשבנות** (סופר אדמין, `/admin/settlements`): לכל ספק — יתרה פתוחה (סכום מחירי העלות של ההזמנות שיוצרו וטרם שולמו), כפתור "יצירת התחשבנות", וסימון "שולם" אחרי ההעברה.
- הספק רואה את אותו מידע לקריאה בלבד במסך ההתחשבנות שלו.

## הרשאות (שליטת סופר אדמין)

לכל צלם, בעריכת המשתמש בניהול:
- **הזמנה מהספק** — האם הצלם רשאי להזמין (זרימות 1 ו-3).
- **לקוחות יכולים להזמין** — האם החנות (Flow B) פעילה ללקוחות של הצלם. אם כבוי — לשונית החנות פשוט לא מופיעה ללקוחות שלו.

## קבלות וחשבוניות (מסמכים)

**אתה עוסק פטור** (ת.ז 305126252) → המערכת מוציאה **קבלה ללא מע"מ**. ניתן להחליף בהגדרה לחשבונית מס/קבלה + מע"מ 18% ביום שתהפוך לעוסק מורשה, בלי שינוי קוד. הפלטפורמה היא המנפיקה בכל המקרים.

| אירוע | מסמך | מתי |
|---|---|---|
| לקוח שילם בחנות | **קבלה** ללקוח | מיידי |
| צלם מבצע הזמנה | **אישור הזמנה** (לא מסמך מס) | מיידי |
| חיוב חודשי לצלם | **קבלה** שמכסה את החודש | בחיוב הכרטיס |
| תשלום מנוי/תוכנית | **קבלה** | בכל חיוב |

- כל מסמך נשלח במייל לנמען ונשמר במערכת.
- הצלם רואה את המסמכים שלו במסך החיוב; הלקוח מקבל קישור "הורדת קבלה" בעמוד הסטטוס.

**מצב נוכחי (חשוב):** הנפקת המסמכים דרך PayPlus **עדיין לא מופעלת**. לכן המערכת רצה ב"מצב מעטפת": כל מסמך **נרשם כ"ממתין"** ונשלח מייל אישור זמני. ברגע ש-PayPlus יפעיל את מודול המסמכים — מפעילים דגל אחד (`PAYPLUS_DOCUMENTS_ENABLED=true`), לוחצים "הנפק מסמכים ממתינים", וכל הקבלות הופכות למסמכים רשמיים ממוספרים עם PDF, ומכאן זה אוטומטי.

## מה נדרש כדי לסיים (להפעלה מלאה)
1. לוודא מול PayPlus שמודול **הנפקת המסמכים** מופעל בחשבון/טרמינל + לאשר את שמות ה-API המדויקים.
2. לוודא ש**charge-by-token** (חיוב כרטיס שמור) מופעל בטרמינל — נדרש לחיוב החודשי האוטומטי.
3. למלא בהגדרות: שם העסק וכתובת, ולהדליק `PAYPLUS_DOCUMENTS_ENABLED=true`.
